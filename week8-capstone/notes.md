## Azure Monitor & Log Analytics
- Collect metrics and logs for VMs, App Services, and other resources.
- Use Log Analytics workspace for centralized query and analysis.

## Alerts
- Create actionable alerts for key metrics (CPU, availability).
- Use Action Groups for notifications (email, webhook).
- Avoid alert fatigue: prioritize critical signals.

## Budgets & Cost Management
- Set monthly budgets with threshold alerts.
- Monitor spend regularly; use tags for cost attribution.
- Combine budgets with lifecycle rules for cost optimization.

## Minimal SLOs
- Define basic availability and performance targets for small apps.
- Example: 99% uptime, CPU < 80%.

## Key Takeaways
- Observability ensures reliability and proactive issue detection.
- Cost controls prevent overspend and enforce governance.
